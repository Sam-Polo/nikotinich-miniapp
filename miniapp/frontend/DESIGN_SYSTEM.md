## Компоненты

### `Button`
Варианты: `primary` (синяя), `secondary` (серая), `ghost` (прозрачная), `danger` (красная).
Пропы: `variant`, `loading`, `fullWidth`.

### `ProductCard`
Карточка товара: фото, название, цена, кнопка «В корзину», иконка избранного.
Пропы: `product`, `showAddButton`.

### `BottomSheet`
Нижний шит с затемнением, drag-индикатором и скроллом.
Пропы: `open`, `onClose`, `title`, `children`, `snapHeight` (по умолчанию `75vh`).

### `SelectionList`
Список выбора (бренд/линейка/модель). Два режима: `list` (с иконкой слева) и `grid` (сетка 3 колонки).
Пропы: `items`, `selected`, `onSelect`, `layout`.

### `PageHeader`
Заголовок страницы в стиле iOS: «Назад» слева, название + подпись по центру, слот справа.
Пропы: `title`, `subtitle`, `showBack`, `right`.

### `BottomNav`
Нижняя навигация: 5 вкладок (Каталог / Новости / Избранное / Корзина / Профиль).
Активная вкладка — синяя, неактивная — серая.
Бейдж с количеством на иконке Корзины.

### `Spinner`
Индикатор загрузки — синий крутящийся кружок по центру.

## Экраны

### Каталог (`/`)
Сетка 2×N с категориями. Квадратные карточки с фото и названием снизу.

### Страница категории (`/catalog/:categoryKey`)
Пошаговый выбор: Бренд → Линейка (или Модель) → Список товаров.
- Если у категории нет брендов — сразу показываются товары.
- Если у бренда нет линеек — сразу показываются товары бренда.
- Выбор через `SelectionList` (список для брендов, сетка для линеек).
- Фильтры через `BottomSheet`.

### Страница товара (`/product/:slug`)
- Большое фото с галереей (dots-индикатор)
- Название, цена (зачёркнутая при скидке)
- Описание с кнопкой «Дальше»
- Характеристики (бренд, линейка, крепость, артикул)
- Кнопка «Добавить в корзину» — sticky снизу
- Иконка сердца в хэдере

### Новости (`/news`)
- Горизонтальный скролл акций (синие плитки)
- Лента новостей (вертикально, фото + текст)
- Подборки товаров (горизонтальный скролл)

### Избранное (`/favorites`)
- Сетка 2×N с товарами
- Кнопка «Добавить всё в корзину»
- Кнопка «Очистить» в хэдере

### Корзина (`/cart`)
- Список товаров: фото, название, цена, счётчик qty, кнопка удалить
- Поле промокода с кнопкой «Применить»
- Итоговый блок: товары, доставка, скидка, итого
- Бесплатная доставка при заказе ≥ `freeDeliveryFrom` ₽

### Оформление заказа (`/checkout`)
- Поля: имя (обязательно), телефон, адрес, комментарий
- Итоги (как в корзине)
- Кнопка «Подтвердить заказ — ₽XXX»

### Успешный заказ (`/order-success/:orderId`)
- Зелёная галочка, номер заказа, сообщение
- Кнопки: «На главную», «Мои заказы»

### Профиль (`/profile`)
- Аватар (инициал) и username
- Вкладки: «Настройки» и «Мои заказы»
- Настройки: поля телефон / email, кнопка сохранить
- Реферальная программа: баланс, ссылка, кнопка скопировать
- История заказов: карточки с номером, датой, статусом, составом

/product/:slug     — карточка товара
/news              — лента новостей и акций
/favorites         — избранное

## Telegram Bot

Бот (`miniapp/bot/`) — точка входа в мини-апп.

| Команда   | Действие                                   |
|-----------|--------------------------------------------|
| `/start`  | Приветствие + кнопка WebApp «Открыть магазин» |
| `/shop`   | То же, без приветствия                     |
| `/news`   | Кнопка с переходом в ленту                 |
| `/help`   | Справка по командам                        |

**Кнопка меню** (у поля ввода) установлена через `setChatMenuButton` с типом `web_app` — открывает мини-апп одним тапом без ввода команды.

**Реферальный deep link:** `https://t.me/<BOT_USERNAME>?startapp=ref_<TELEGRAM_ID>`.  
Telegram передаёт `start_param = "ref_<TELEGRAM_ID>"` → бот отправляет приветствие с пометкой о приглашении → мини-апп читает `initDataUnsafe.start_param` и передаёт `referrer_id` при регистрации пользователя.

**ENV:**
- `BOT_TOKEN` — токен из @BotFather
- `MINIAPP_URL` — публичный https URL фронтенда (Telegram требует https для WebApp)
- `VITE_BOT_USERNAME` — username бота (фронтенд, для формирования реферальной ссылки)

## Telegram SDK

- `WebApp.ready()` + `WebApp.expand()` при монтировании App.
- Пользователь: `WebApp.initDataUnsafe.user.id` → `telegram_id`.
- Реферер: `start_param` вида `ref_<telegram_id>`.
- `VITE_BOT_USERNAME` (env) — username бота для формирования реферальной ссылки.
